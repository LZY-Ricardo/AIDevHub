# AIDevHub (MVP) 后端接口文档（给前端）

> 形态：Tauri v2 桌面应用，前端通过 **Tauri commands（IPC）** 调用 Rust 后端。
> 目标：覆盖 P0（Servers 管理、Profile、Diff 预览、备份与回滚）。

## 1. 通用约定

### 1.1 调用与返回

- 所有命令均为 `tauri::command`，前端通过 `invoke(command, payload)` 调用。
- 命令返回 `Result<T, AppError>`：
  - `Ok(T)`：前端 Promise resolve
  - `Err(AppError)`：前端 Promise reject  
  以上行为与 Tauri v2 的 `invoke`/`Result` 约定一致（我们已通过 Tauri v2 文档核对）。

### 1.2 双阶段写入（重要）

任何会改写文件的操作都分两步：

1. `*_preview_*`：只读，不落盘；返回 `WritePreview`（包含 diff、摘要、预期前置条件）。
2. `*_apply_*`：真正写入；必须携带 `expected_files`（来自 preview），用于检测配置文件是否在确认后被外部修改。

若前置条件不满足，返回 `AppError{ code: "PRECONDITION_FAILED" }`，前端需要重新 preview。

### 1.3 ServerId 约定

- `server_id = "<client>:<name>"`  
  例：`claude_code:github`、`codex:context7`

### 1.4 Claude 与 Codex 的启用/禁用策略差异

- `claude_code`：配置里没有通用 `enabled` 字段，MVP 采用“禁用池（disabled_pool.json）”：
  - 禁用：从 `~/.claude.json` 的 `mcpServers` 移除，写入禁用池
  - 启用：从禁用池写回 `mcpServers`
- `codex`：只改 `~/.codex/config.toml` 中 `mcp_servers.<name>.enabled = true/false`，不删除配置块

## 2. 数据结构（TypeScript 视角）

```ts
export type Client = "claude_code" | "codex";
export type Transport = "stdio" | "http" | "unknown";

export interface AppError {
  code:
    | "VALIDATION_ERROR"
    | "NOT_FOUND"
    | "PARSE_ERROR"
    | "IO_ERROR"
    | "PRECONDITION_FAILED"
    | "UNSUPPORTED"
    | "INTERNAL_ERROR";
  message: string;
  details?: unknown;
}

export interface Warning {
  code: "MISSING_SERVER" | "HIGH_RISK" | "ENV_VAR_MISSING" | "SKIPPED";
  message: string;
  details?: unknown;
}

export interface ServerRecord {
  server_id: string;     // "<client>:<name>"
  name: string;          // 原始名称（Claude 的 mcpServers key / Codex 的 table id）
  client: Client;
  transport: Transport;  // stdio/http（无法识别则 unknown）
  enabled: boolean;
  source_file: string;   // 例：/home/user/.claude.json
  identity: string;      // stdio: "cmd args..." / http: "url"
  payload: Record<string, unknown>; // 详情展示用（默认会对敏感值做掩码）
}

export interface Profile {
  profile_id: string;
  name: string;
  targets: {
    claude_code: string[]; // server_id 列表
    codex: string[];       // server_id 列表
  };
  updated_at: string; // ISO8601
}

export interface FilePrecondition {
  path: string;                 // 将要写入的目标路径
  expected_before_sha256: string | null; // preview 阶段的 before；null 表示当时文件不存在
}

export interface FileChangePreview {
  path: string;
  will_create: boolean;
  before_sha256: string | null;
  after_sha256: string;
  diff_unified: string; // unified diff（可能较长）
}

export interface WriteSummary {
  will_enable: string[];  // server_id
  will_disable: string[]; // server_id
  will_add: string[];     // server_id（新增 server）
}

export interface WritePreview {
  files: FileChangePreview[];
  summary: WriteSummary;
  warnings: Warning[];
}

export type BackupOp = "toggle" | "add_server" | "apply_profile" | "rollback";

export interface BackupRecord {
  backup_id: string;
  target_path: string;
  backup_path: string;
  created_at: string; // ISO8601
  op: BackupOp;
  summary: string;
}

export interface ApplyResult {
  backups: BackupRecord[]; // 本次写入涉及的备份记录
  summary: WriteSummary;
  warnings: Warning[];
}
```

## 3. Commands（P0）

### 3.1 运行时信息

#### `runtime_get_info`

用途：前端展示“来源文件路径/备份目录”等信息，便于用户理解与排错。

请求：`{}`

响应：

```ts
{
  paths: {
    claude_config_path: string; // "~/.claude.json" 的解析结果（绝对路径）
    codex_config_path: string;  // "~/.codex/config.toml" 或 CODEX_HOME/config.toml（绝对路径）
    app_local_data_dir: string; // AIDevHub 应用数据目录（Local 语义）
    profiles_path: string;
    disabled_pool_path: string;
    backups_dir: string;
    backup_index_path: string;
  };
  exists: {
    claude_config: boolean;
    codex_config: boolean;
  };
}
```

### 3.2 Servers

#### `server_list`

用途：读取并展示 servers 列表（Claude + Codex），包含启用状态（Claude 会合并禁用池）。

请求：

```ts
{ client?: Client } // 不传则返回全部
```

响应：`ServerRecord[]`

#### `server_get`

用途：打开详情抽屉时读取单个 server（可选揭示敏感值）。

请求：

```ts
{
  server_id: string;
  reveal_secrets?: boolean; // 默认 false
}
```

响应：`ServerRecord`

#### `server_preview_toggle`

用途：单个 server 开关的变更预览（生成 diff，不写入）。

请求：

```ts
{
  server_id: string;
  enabled: boolean; // 期望目标状态
}
```

响应：`WritePreview`

#### `server_apply_toggle`

用途：执行单个 server 开关（会备份并写入）。

请求：

```ts
{
  server_id: string;
  enabled: boolean;
  expected_files: FilePrecondition[]; // 必须来自对应 preview
}
```

响应：`ApplyResult`

#### `server_preview_add`

用途：新增 server 的变更预览（生成 diff，不写入）。

请求（MVP 表单输入）：

```ts
{
  client: Client;
  name: string; // server 名称（Claude 的 key / Codex 的 table id）
  transport: "stdio" | "http";
  // config 为“该客户端配置片段”，MVP 只做最小校验，其它字段透传写入
  // claude_code + stdio: { command, args?, env? }
  // claude_code + http:  { url, headers? }（后端会补 type="http"）
  // codex + stdio:       { command, args? }（后端会强制 enabled=true）
  // codex + http:        { url, bearer_token_env_var? }（后端会强制 enabled=true）
  config: Record<string, unknown>;
}
```

响应：`WritePreview`

#### `server_apply_add`

用途：执行新增 server（会备份并写入）。

请求：

```ts
{
  client: Client;
  name: string;
  transport: "stdio" | "http";
  config: Record<string, unknown>;
  expected_files: FilePrecondition[];
}
```

响应：`ApplyResult`

### 3.3 Profiles

#### `profile_list`

用途：Profile 列表读取。

请求：`{}`

响应：`Profile[]`

#### `profile_create`

用途：新建 Profile。

请求：

```ts
{
  name: string;
  targets: Profile["targets"];
}
```

响应：`Profile`

#### `profile_update`

用途：编辑 Profile（名称、targets）。

请求：

```ts
{
  profile_id: string;
  name?: string;
  targets?: Profile["targets"];
}
```

响应：`Profile`

#### `profile_delete`

用途：删除 Profile。

请求：

```ts
{ profile_id: string }
```

响应：

```ts
{ ok: true }
```

#### `profile_preview_apply`

用途：将 Profile “收敛模式”应用到单个客户端的预览（生成 diff，不写入）。

请求：

```ts
{
  profile_id: string;
  client: Client; // MVP：一次只应用一个客户端；需要双端就调用两次
}
```

响应：`WritePreview`

说明（MVP 关键行为）：

- 应用后：目标客户端的启用集合 **精确等于** Profile.targets[client]
- Profile 引用缺失的 server：在 `warnings` 里返回 `MISSING_SERVER`，并跳过该 server（不阻断）

#### `profile_apply`

用途：执行 Profile 应用（会备份并写入）。

请求：

```ts
{
  profile_id: string;
  client: Client;
  expected_files: FilePrecondition[];
}
```

响应：`ApplyResult`

### 3.4 Backups / Rollback

#### `backup_list`

用途：列出备份历史（用于回滚 UI）。

请求：

```ts
{
  target_path?: string; // 可选：只筛某个文件（如 claude_config_path）
}
```

响应：`BackupRecord[]`

#### `backup_preview_rollback`

用途：回滚预览（生成 diff，不写入）。

请求：

```ts
{
  backup_id: string;
}
```

响应：`WritePreview`

说明：preview 会以“当前目标文件内容” vs “备份内容”生成 diff；apply 时会先再次备份当前文件，再恢复目标备份。

#### `backup_apply_rollback`

用途：执行回滚（会产生新的备份记录，并写入恢复内容）。

请求：

```ts
{
  backup_id: string;
  expected_files: FilePrecondition[]; // 至少包含目标文件 path 的 precondition
}
```

响应：`ApplyResult`
