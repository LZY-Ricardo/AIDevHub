# AIDevHub (MVP) 后端接口文档（给前端）

> 形态：Tauri v2 桌面应用，前端通过 **Tauri commands（IPC）** 调用 Rust 后端。
> 目标：覆盖 P0（Servers 管理、编辑、Skills、Profile、Diff 预览、备份与回滚）。
> 同步基准：commands 注册见 `app-v2/src-tauri/src/lib.rs`；数据结构见 `app-v2/src-tauri/crates/aidevhub-core/src/model.rs`。

## 1. 通用约定

### 1.1 调用与返回

- 所有命令均为 `tauri::command`，前端通过 `invoke(command, payload)` 调用。
- 命令返回 `Result<T, AppError>`：
  - `Ok(T)`：前端 Promise resolve
  - `Err(AppError)`：前端 Promise reject  
  以上行为与 Tauri v2 的 `invoke`/`Result` 约定一致（我们已通过 Tauri v2 文档核对）。

### 1.2 双阶段写入（重要）

任何会改写文件的操作都分两步：

1. `*_preview_*`：只读，不落盘；返回 `WritePreview`（包含 diff、摘要、预期前置条件）。
2. `*_apply_*`：真正写入；必须携带 `expected_files`（来自 preview），用于检测配置文件是否在确认后被外部修改。

若前置条件不满足，返回 `AppError{ code: "PRECONDITION_FAILED" }`，前端需要重新 preview。

### 1.3 ServerId 约定

- `server_id = "<client>:<name>"`  
  例：`claude_code:github`、`codex:context7`

### 1.4 Claude 与 Codex 的启用/禁用策略差异

- `claude_code`：配置里没有通用 `enabled` 字段，MVP 采用“禁用池（disabled_pool.json）”：
  - 禁用：从 `~/.claude.json` 的 `mcpServers` 移除，写入禁用池
  - 启用：从禁用池写回 `mcpServers`
- `codex`：只改 `~/.codex/config.toml` 中 `mcp_servers.<name>.enabled = true/false`，不删除配置块

## 2. 数据结构（TypeScript 视角）

```ts
export type Client = "claude_code" | "codex";
export type Transport = "stdio" | "http" | "unknown";

export interface AppError {
  // 与 app-v2/src/lib/types.ts 对齐。当前后端主要返回 VALIDATION/NOT_FOUND/PARSE/IO/PRECONDITION/INTERNAL。
  code:
    | "VALIDATION_ERROR"
    | "NOT_FOUND"
    | "PARSE_ERROR"
    | "IO_ERROR"
    | "PRECONDITION_FAILED"
    | "UNSUPPORTED"
    | "INTERNAL_ERROR";
  message: string;
  details?: unknown;
}

export interface Warning {
  // 与 app-v2/src/lib/types.ts 对齐；当前后端实际主要返回 "MISSING_SERVER"。
  code: "MISSING_SERVER" | "HIGH_RISK" | "ENV_VAR_MISSING" | "SKIPPED";
  message: string;
  details?: unknown;
}

export interface ServerRecord {
  server_id: string;     // "<client>:<name>"
  name: string;          // 原始名称（Claude 的 mcpServers key / Codex 的 table id）
  client: Client;
  transport: Transport;  // stdio/http（无法识别则 unknown）
  enabled: boolean;
  source_file: string;   // 例：/home/user/.claude.json
  identity: string;      // stdio: "cmd args..." / http: "url"
  payload: Record<string, unknown>; // 详情展示用（默认会对敏感值做掩码）
}

export interface ServerFieldMeta {
  known_fields: string[];
  secret_fields: string[];
  readonly_fields: string[];
  available_fields: string[];
}

export interface ServerEditSession {
  server_id: string;
  client: Client;
  transport: "stdio" | "http";
  source_file: string;
  editable_payload: Record<string, unknown>;
  raw_fragment_json: Record<string, unknown>;
  unknown_fields: string[];
  field_meta: ServerFieldMeta;
}

export interface ServerNotes {
  description: string;
  field_hints: Record<string, string>;
}

export interface Profile {
  profile_id: string;
  name: string;
  targets: {
    claude_code: string[]; // server_id 列表
    codex: string[];       // server_id 列表
  };
  updated_at: string; // ISO8601
}

export interface FilePrecondition {
  path: string;                 // 将要写入的目标路径
  expected_before_sha256: string | null; // preview 阶段的 before；null 表示当时文件不存在
}

export interface FileChangePreview {
  path: string;
  will_create: boolean;
  before_sha256: string | null;
  after_sha256: string;
  diff_unified: string; // unified diff（可能较长）
}

export interface WriteSummary {
  will_enable: string[];  // id（server_id / skill_id）
  will_disable: string[]; // id（server_id / skill_id）
  will_add: string[];     // id（新增 server / 新增 skill）
}

export type SkillScope = "user" | "system";
export type SkillKind = "dir" | "file";

export interface MovePreview {
  from: string;
  to: string;
  kind: SkillKind; // "dir" | "file"
}

export interface WritePreview {
  files: FileChangePreview[];
  moves?: MovePreview[];              // 可选：目录/文件移动预览（skills toggle 会用到）
  expected_files?: FilePrecondition[]; // 可选：预览生成的前置条件（优先用于 apply）
  summary: WriteSummary;
  warnings: Warning[];
}

export interface SkillRecord {
  skill_id: string; // "<client>:<name>"
  client: Client;
  name: string;
  description: string;
  scope: SkillScope;
  kind: SkillKind;
  enabled: boolean;
  entry_path: string;
  container_path: string;
}

export interface SkillGetResponse {
  record: SkillRecord;
  content: string; // entry_path 的原文
}

export type BackupOp = "toggle" | "add_server" | "edit_server" | "apply_profile" | "rollback";

export interface BackupRecord {
  backup_id: string;
  target_path: string;
  backup_path: string;
  created_at: string; // ISO8601
  op: BackupOp;
  summary: string;
}

export interface ApplyResult {
  backups: BackupRecord[]; // 本次写入涉及的备份记录
  summary: WriteSummary;
  warnings: Warning[];
}
```

## 3. Commands（P0）

### 3.1 运行时信息

#### `runtime_get_info`

用途：前端展示“来源文件路径/备份目录”等信息，便于用户理解与排错。

请求：`{}`

响应：

```ts
{
  paths: {
    claude_config_path: string; // "~/.claude.json" 的解析结果（绝对路径）
    claude_commands_dir: string; // "~/.claude/commands"（绝对路径）
    claude_commands_disabled_dir: string; // "~/.claude/commands_disabled"（绝对路径）
    codex_config_path: string;  // "~/.codex/config.toml" 或 CODEX_HOME/config.toml（绝对路径）
    codex_skills_dir: string; // "<codex_home>/skills"（绝对路径）
    codex_skills_disabled_dir: string; // "<codex_home>/skills_disabled"（绝对路径）
    app_local_data_dir: string; // AIDevHub 应用数据目录（Local 语义）
    profiles_path: string;
    disabled_pool_path: string;
    backups_dir: string;
    backup_index_path: string;
  };
  exists: {
    claude_config: boolean;
    codex_config: boolean;
  };
}
```

### 3.2 Servers

#### `server_list`

用途：读取并展示 servers 列表（Claude + Codex），包含启用状态（Claude 会合并禁用池）。

请求：

```ts
{ client?: Client } // 不传则返回全部
```

响应：`ServerRecord[]`

#### `server_get`

用途：打开详情抽屉时读取单个 server（可选揭示敏感值）。

请求：

```ts
{
  server_id: string;
  reveal_secrets?: boolean; // 默认 false
}
```

响应：`ServerRecord`

#### `server_preview_toggle`

用途：单个 server 开关的变更预览（生成 diff，不写入）。

请求：

```ts
{
  server_id: string;
  enabled: boolean; // 期望目标状态
}
```

响应：`WritePreview`

#### `server_apply_toggle`

用途：执行单个 server 开关（会备份并写入）。

请求：

```ts
{
  server_id: string;
  enabled: boolean;
  expected_files: FilePrecondition[]; // 必须来自对应 preview
}
```

响应：`ApplyResult`

说明：创建 skill/command 不会生成用户配置备份，通常 `backups` 为空数组。

#### `server_get_edit_session`

用途：打开编辑器时读取当前 server 的可编辑会话（包含结构化 payload、原始片段与字段元信息）。

请求：

```ts
{
  server_id: string;
}
```

响应：`ServerEditSession`

说明：

- `raw_fragment_json` 是当前 MCP 的完整 JSON 片段，供高级编辑器直接修改
- `unknown_fields` 标记当前前端表单未结构化处理、但会在保存时保留的附加字段
- `field_meta` 用于渲染“只读字段”“敏感字段”“可编辑字段”说明

#### `server_notes_get`

用途：读取某个 MCP 的人工说明与字段提示。

请求：

```ts
{
  server_id: string;
}
```

响应：`ServerNotes`

说明：

- 若该 MCP 尚未保存人工说明，返回空对象语义：`{ description: "", field_hints: {} }`

#### `server_notes_put`

用途：保存某个 MCP 的人工说明与字段提示。

请求：

```ts
{
  server_id: string;
  notes: ServerNotes;
}
```

响应：`ServerNotes`

说明：

- 数据持久化在应用本地数据目录下的 `mcp_notes.json`
- 空 description 且空 `field_hints` 会被视为删除该 MCP 的人工说明记录

#### `server_preview_edit`

用途：基于编辑草稿生成变更预览（生成 diff，不写入）。

请求：

```ts
{
  server_id: string;
  transport: "stdio" | "http";
  payload: Record<string, unknown>;
}
```

响应：`WritePreview`

#### `server_apply_edit`

用途：执行单个 server 的编辑写入（会备份并写入）。

请求：

```ts
{
  server_id: string;
  transport: "stdio" | "http";
  payload: Record<string, unknown>;
  expected_files: FilePrecondition[]; // 必须来自对应 preview
}
```

响应：`ApplyResult`

说明：启用/禁用 skill/command 通过目录或文件移动完成，不会生成用户配置备份，通常 `backups` 为空数组。

说明：会生成用户配置文件备份，`ApplyResult.backups[*].op === "edit_server"`。

### 3.5 Skills（Codex skills + Claude commands）

说明：

- Codex skills：位于 `<codex_home>/skills/<name>/SKILL.md`，禁用池为 `<codex_home>/skills_disabled/<name>/SKILL.md`
- Claude commands（个人命令）：位于 `~/.claude/commands/<name>.md`，禁用池为 `~/.claude/commands_disabled/<name>.md`
- Codex 的系统 skills（`codex:.system/...`）只读展示，P0 不允许 toggle

#### `skill_list`

用途：列出 skills/commands 清单（含禁用池）。

请求：

```ts
{
  client?: Client; // 可选：只看 claude_code 或 codex
  scope?: "all" | "user" | "system" | "disabled"; // 可选：默认 all
}
```

响应：`SkillRecord[]`

#### `skill_get`

用途：读取单个 skill/command 详情（含入口文件原文）。

请求：

```ts
{
  skill_id: string; // "<client>:<name>"
}
```

响应：`SkillGetResponse`

#### `skill_preview_create`

用途：创建 skill/command 的预览（生成 diff，不写入）。

请求：

```ts
{
  client: Client;
  name: string;         // codex: 目录名；claude_code: 命令名（不含 .md）
  description: string;  // 写入 frontmatter.description
  body?: string;        // 可选：正文（不传则用最小模板）
}
```

响应：`WritePreview`

#### `skill_apply_create`

用途：执行创建（会写入文件；不生成 backups 记录）。

请求：

```ts
{
  client: Client;
  name: string;
  description: string;
  body?: string;
  expected_files: FilePrecondition[]; // 来自 preview（用于防止竞态）
}
```

响应：`ApplyResult`

#### `skill_preview_toggle`

用途：启用/禁用 skill/command 的预览（生成 moves，不写入）。

请求：

```ts
{
  skill_id: string;
  enabled: boolean;
}
```

响应：`WritePreview`（其中 `moves` 会包含移动计划）

#### `skill_apply_toggle`

用途：执行启用/禁用（移动目录或文件；不生成 backups 记录）。

请求：

```ts
{
  skill_id: string;
  enabled: boolean;
  expected_files: FilePrecondition[]; // 来自 preview（用于防止竞态）
}
```

响应：`ApplyResult`

#### `server_preview_add`

用途：新增 server 的变更预览（生成 diff，不写入）。

请求（MVP 表单输入）：

```ts
{
  client: Client;
  name: string; // server 名称（Claude 的 key / Codex 的 table id）
  transport: "stdio" | "http";
  // config 为“该客户端配置片段”，MVP 只做最小校验，其它字段透传写入
  // claude_code + stdio: { command, args?, env? }
  // claude_code + http:  { url, headers? }（后端会补 type="http"）
  // codex + stdio:       { command, args? }（后端会强制 enabled=true）
  // codex + http:        { url, bearer_token_env_var? }（后端会强制 enabled=true）
  config: Record<string, unknown>;
}
```

响应：`WritePreview`

#### `server_apply_add`

用途：执行新增 server（会备份并写入）。

请求：

```ts
{
  client: Client;
  name: string;
  transport: "stdio" | "http";
  config: Record<string, unknown>;
  expected_files: FilePrecondition[];
}
```

响应：`ApplyResult`

### 3.3 Profiles

#### `profile_list`

用途：Profile 列表读取。

请求：`{}`

响应：`Profile[]`

#### `profile_create`

用途：新建 Profile。

请求：

```ts
{
  name: string;
  targets: Profile["targets"];
}
```

响应：`Profile`

#### `profile_update`

用途：编辑 Profile（名称、targets）。

请求：

```ts
{
  profile_id: string;
  name?: string;
  targets?: Profile["targets"];
}
```

响应：`Profile`

#### `profile_delete`

用途：删除 Profile。

请求：

```ts
{ profile_id: string }
```

响应：

```ts
{ ok: true }
```

#### `profile_preview_apply`

用途：将 Profile “收敛模式”应用到单个客户端的预览（生成 diff，不写入）。

请求：

```ts
{
  profile_id: string;
  client: Client; // MVP：一次只应用一个客户端；需要双端就调用两次
}
```

响应：`WritePreview`

说明（MVP 关键行为）：

- 应用后：目标客户端的启用集合 **精确等于** Profile.targets[client]
- Profile 引用缺失的 server：在 `warnings` 里返回 `MISSING_SERVER`，并跳过该 server（不阻断）

#### `profile_apply`

用途：执行 Profile 应用（会备份并写入）。

请求：

```ts
{
  profile_id: string;
  client: Client;
  expected_files: FilePrecondition[];
}
```

响应：`ApplyResult`

### 3.4 Backups / Rollback

#### `backup_list`

用途：列出备份历史（用于回滚 UI）。

请求：

```ts
{
  target_path?: string; // 可选：只筛某个文件（如 claude_config_path）
}
```

响应：`BackupRecord[]`

#### `backup_preview_rollback`

用途：回滚预览（生成 diff，不写入）。

请求：

```ts
{
  backup_id: string;
}
```

响应：`WritePreview`

说明：preview 会以“当前目标文件内容” vs “备份内容”生成 diff；apply 时会先再次备份当前文件，再恢复目标备份。

#### `backup_apply_rollback`

用途：执行回滚（会产生新的备份记录，并写入恢复内容）。

请求：

```ts
{
  backup_id: string;
  expected_files: FilePrecondition[]; // 至少包含目标文件 path 的 precondition
}
```

响应：`ApplyResult`
