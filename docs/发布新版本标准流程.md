# AIDevHub 发布新版本标准流程

**最后验证日期**：2026-04-20
**适用范围**：AIDevHub 桌面端 Tauri 发布、GitHub Release、应用内更新元数据上传
**当前已验证版本**：`0.4.2`

---

## 1. 目标

这份文档记录 AIDevHub 在当前仓库下发布新版本的正确流程，避免重复踩坑。

当前推荐主路径：

- [release.yml](/F:/myProjects/AIDevHub/.github/workflows/release.yml)

也就是：

- 先把版本号提交到 `main`
- 再通过 GitHub Actions 手动触发正式发布

当前发布目标包括：

- 生成并上传 Windows 安装包
- 生成并上传签名文件
- 生成并上传 `v1Compatible` updater zip 包
- 生成并上传 `latest.json`
- 创建新的 GitHub Release

## 2. 当前发布模式

当前项目使用：

- Tauri v2
- updater 插件
- GitHub Release 作为更新分发源
- `latest.json` 作为静态更新元数据

关键配置文件：

- [package.json](/F:/myProjects/AIDevHub/app-v2/package.json)
- [Cargo.toml](/F:/myProjects/AIDevHub/app-v2/src-tauri/Cargo.toml)
- [tauri.conf.json](/F:/myProjects/AIDevHub/app-v2/src-tauri/tauri.conf.json)

当前关键配置：

```json
"bundle": {
  "active": true,
  "targets": "all",
  "createUpdaterArtifacts": "v1Compatible"
},
"plugins": {
  "updater": {
    "endpoints": [
      "https://github.com/LZY-Ricardo/AIDevHub/releases/latest/download/latest.json"
    ]
  }
}
```

结论：

- Windows 下必须继续用 `v1Compatible`
- 因为当前客户端更新端点依赖 `latest.json`

## 4. 标准发布步骤（推荐）

### Step 1：在 `main` 上提升版本号

使用内置脚本一键同步修改以下四个文件中的版本号：

- `package.json`
- `Cargo.toml`
- `Cargo.lock`
- `tauri.conf.json`

```bash
# 示例：从 0.4.2 升级到 0.4.3
node -e "import { updateReleaseVersion } from './app-v2/scripts/update-release-version.mjs'; updateReleaseVersion('app-v2', '0.4.3')"
```

验证版本号是否一致：

```bash
node -e "import { readReleaseVersions } from './app-v2/scripts/update-release-version.mjs'; console.log(JSON.stringify(readReleaseVersions('app-v2'), null, 2))"
```

四个字段应该全部显示 `0.4.3`。

提交并推送：

```bash
git add app-v2/package.json app-v2/src-tauri/Cargo.toml app-v2/src-tauri/tauri.conf.json app-v2/src-tauri/Cargo.lock
git commit -m "chore(release): 升级版本到0.4.3"
git push origin main
```

### Step 2：在 GitHub Actions 中手动触发 `Release Automation`

**方式 A：命令行触发（推荐）**

```bash
gh workflow run release.yml --ref main \
  -f version=0.4.3 \
  -f notes="本次发布的更新说明"
```

**方式 B：浏览器触发**

1. 打开 `https://github.com/LZY-Ricardo/AIDevHub/actions/workflows/release.yml`
2. 点击 **Run workflow**
3. 填入：
   - `version`：例如 `0.4.3`（必须与 Step 1 中升级的版本一致）
   - `notes`：发布说明（支持中文）

输入参数说明：

- `version`：纯数字版本号，例如 `0.4.3`
- `notes`：发布说明，支持中文和特殊字符

workflow 会自动完成：

- 只允许从 `main` 触发
- 校验版本文件和输入版本一致
- 如果同版本 `tag/release` 已存在，直接 fail-fast
- 如果创建 release 中途失败，自动清理本次新建的 release/tag
- 生成签名安装包
- 生成 `v1Compatible` updater zip 包
- 校验产物完整性
- 生成并校验 `latest.json`
- 创建全新的 GitHub Release

### Step 3：监控发布进度

```bash
# 查看运行状态（替换为实际的 run ID）
gh run list --workflow=release.yml --limit=1

# 实时跟踪
gh run watch <run-id> --exit-status
```

整个流程约 10-15 分钟（主要是 Tauri 构建耗时）。

### Step 4：发布后核验

```bash
gh release view app-v0.4.3 --json assets --jq '{assetCount: (.assets | length), assets: [.assets[].name]}'
```

确认 Release 包含以下 9 个资产：

| 资产 | 说明 |
|------|------|
| `AIDevHub_0.4.3_x64_en-US.msi` | MSI 安装包 |
| `AIDevHub_0.4.3_x64_en-US.msi.sig` | MSI 签名 |
| `AIDevHub_0.4.3_x64_en-US.msi.zip` | MSI 更新包 |
| `AIDevHub_0.4.3_x64_en-US.msi.zip.sig` | MSI 更新包签名 |
| `AIDevHub_0.4.3_x64-setup.exe` | NSIS 安装包 |
| `AIDevHub_0.4.3_x64-setup.exe.sig` | NSIS 签名 |
| `AIDevHub_0.4.3_x64-setup.nsis.zip` | NSIS 更新包 |
| `AIDevHub_0.4.3_x64-setup.nsis.zip.sig` | NSIS 更新包签名 |
| `latest.json` | 自动更新元数据 |

## 5. 一键发布速查

完整流程浓缩为四条命令（假设发布 `0.4.3`）：

```bash
# 1. 升级版本号
node -e "import { updateReleaseVersion } from './app-v2/scripts/update-release-version.mjs'; updateReleaseVersion('app-v2', '0.4.3')"

# 2. 提交推送
git add app-v2/package.json app-v2/src-tauri/Cargo.toml app-v2/src-tauri/tauri.conf.json app-v2/src-tauri/Cargo.lock
git commit -m "chore(release): 升级版本到0.4.3" && git push

# 3. 触发发布
gh workflow run release.yml --ref main -f version=0.4.3 -f notes="发布说明"

# 4. 查看结果
gh run list --workflow=release.yml --limit=1
```

## 7. 手工兜底发布（仅在 workflow 不可用时）

只有在 workflow 无法使用时，才建议走手工兜底流程。

手工兜底至少需要：

1. 本地设置签名环境变量
2. 执行 `pnpm tauri build`
3. 手工生成 `latest.json`
4. 手工创建 release 并上传全部资产

手工命令示例：

```bash
cd app-v2
export TAURI_SIGNING_PRIVATE_KEY="$HOME/.tauri/aidevhub/updater.key"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD=""
pnpm tauri build
```

然后在仓库根目录：

```bash
gh release create app-v0.4.3 ...
gh release view app-v0.4.3 --json assets
```

## 8. `latest.json` 的格式

当前项目使用的 `latest.json` 格式：

```json
{
  "version": "0.4.3",
  "notes": "发布说明",
  "pub_date": "2026-04-20T10:00:00.0000000+08:00",
  "platforms": {
    "windows-x86_64": {
      "signature": "<nsis zip 的 .sig 文件内容>",
      "url": "https://github.com/LZY-Ricardo/AIDevHub/releases/download/app-v0.4.3/AIDevHub_0.4.3_x64-setup.nsis.zip"
    }
  }
}
```

当前已验证：

- `signature` 来自 `*.nsis.zip.sig`
- `url` 指向同版本 release 下的 `*.nsis.zip`
- 目前只覆盖 `windows-x86_64`

## 10. 当前已知限制

1. 当前 workflow 只覆盖 Windows `windows-x86_64`
2. `com.aidevhub.app` 这个 bundle identifier 以 `.app` 结尾，Tauri 会给出警告，后续建议调整
3. 手工发布流程仅作为兜底，不应再作为默认路径
