# AIDevHub 发布新版本标准流程

**最后验证日期**：2026-04-19  
**适用范围**：AIDevHub 桌面端 Tauri 发布、GitHub Release、应用内更新元数据上传  
**当前已验证版本**：`0.4.1`

---

## 1. 目标

这份文档记录 AIDevHub 在当前仓库下发布新版本时的正确动作顺序，避免重复踩坑。

当前推荐优先使用：

- [release.yml](/F:/myProjects/AIDevHub/.github/workflows/release.yml)

也就是通过 GitHub Actions 手动触发发布，而不是完全依赖本地终端命令。

当前项目的发布目标包括：

- 推送 `main` 到远端
- 创建新的 GitHub Release
- 上传 Windows 安装包
- 上传签名文件
- 上传兼容 updater 的 zip 包
- 生成并上传 `latest.json`

---

## 2. 当前发布模式

当前项目使用：

- Tauri v2
- updater 插件
- GitHub Release 作为更新分发源
- `latest.json` 作为静态更新元数据

当前配置文件：

- [package.json](/F:/myProjects/AIDevHub/app-v2/package.json)
- [Cargo.toml](/F:/myProjects/AIDevHub/app-v2/src-tauri/Cargo.toml)
- [tauri.conf.json](/F:/myProjects/AIDevHub/app-v2/src-tauri/tauri.conf.json)

当前关键配置：

```json
"bundle": {
  "active": true,
  "targets": "all",
  "createUpdaterArtifacts": "v1Compatible"
},
"plugins": {
  "updater": {
    "endpoints": [
      "https://github.com/LZY-Ricardo/AIDevHub/releases/latest/download/latest.json"
    ]
  }
}
```

注意：

- `createUpdaterArtifacts: true` 在 Windows 下不会自动产出 `latest.json`
- 当前项目为了兼容现有 `latest.json` 分发方式，必须使用：
  - `"createUpdaterArtifacts": "v1Compatible"`

## 3. 必要前提

发布前必须确认：

1. 本地代码已经合并到 `main`
2. `pnpm build` 可通过
3. GitHub CLI 已登录：

```powershell
gh auth status
```

4. updater 私钥存在且与 `tauri.conf.json` 中的 `pubkey` 匹配

当前已验证可用的密钥路径：

```text
C:\Users\lenovo\.tauri\aidevhub\updater.key
C:\Users\lenovo\.tauri\aidevhub\updater.key.pub
```

## 4. 发布步骤

### Step 1：升级版本号

同步修改以下文件中的版本号，例如从 `0.4.1` 升级到 `0.4.2`：

- [package.json](/F:/myProjects/AIDevHub/app-v2/package.json)
- [Cargo.toml](/F:/myProjects/AIDevHub/app-v2/src-tauri/Cargo.toml)
- [tauri.conf.json](/F:/myProjects/AIDevHub/app-v2/src-tauri/tauri.conf.json)

必要时同步锁文件中的根包版本：

- [Cargo.lock](/F:/myProjects/AIDevHub/app-v2/src-tauri/Cargo.lock)

推荐版本提交：

```text
chore(release): 升级版本到0.4.x
```

---

### Step 2：本地验证前端构建

在 [app-v2](/F:/myProjects/AIDevHub/app-v2) 目录执行：

```powershell
pnpm build
```

预期：

- TypeScript 编译通过
- Vite 构建成功

---

### Step 3：设置 updater 私钥环境变量

在同一个 PowerShell 会话中执行：

```powershell
$env:TAURI_SIGNING_PRIVATE_KEY="$HOME/.tauri/aidevhub/updater.key"
$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD=""
```

如果私钥有密码，把第二行改成真实密码。

注意：

- `.env` 文件无效
- 必须在执行 `pnpm tauri build` 的同一个终端会话里设置

---

### Step 4：构建安装包与 updater 兼容产物

在 [app-v2](/F:/myProjects/AIDevHub/app-v2) 目录执行：

```powershell
pnpm tauri build
```

当前已验证的 Windows 产物：

- 安装包
  - `app-v2/src-tauri/target/release/bundle/msi/AIDevHub_<version>_x64_en-US.msi`
  - `app-v2/src-tauri/target/release/bundle/nsis/AIDevHub_<version>_x64-setup.exe`
- 安装包签名
  - `*.msi.sig`
  - `*.exe.sig`
- v1Compatible updater 兼容包
  - `*.msi.zip`
  - `*.msi.zip.sig`
  - `*.nsis.zip`
  - `*.nsis.zip.sig`

关键认知：

- `v1Compatible` 会生成 updater 用的 zip 包
- 但不会自动帮你生成 GitHub Release 需要的 `latest.json`

---

### Step 5：推送 `main`

在仓库根目录执行：

```powershell
git push origin main
```

---

### Step 6：创建 GitHub Release

示例：

```powershell
gh release create app-v0.4.1 `
  "app-v2/src-tauri/target/release/bundle/msi/AIDevHub_0.4.1_x64_en-US.msi" `
  "app-v2/src-tauri/target/release/bundle/nsis/AIDevHub_0.4.1_x64-setup.exe" `
  --target main `
  --title "AIDevHub v0.4.1" `
  --notes "发布说明"
```

如果 release 已存在，则改用：

```powershell
gh release upload app-v0.4.1 <files...> --clobber
```

---

### Step 7：上传签名文件与 updater 兼容包

示例：

```powershell
gh release upload app-v0.4.1 `
  "app-v2/src-tauri/target/release/bundle/msi/AIDevHub_0.4.1_x64_en-US.msi.sig" `
  "app-v2/src-tauri/target/release/bundle/nsis/AIDevHub_0.4.1_x64-setup.exe.sig" `
  "app-v2/src-tauri/target/release/bundle/msi/AIDevHub_0.4.1_x64_en-US.msi.zip" `
  "app-v2/src-tauri/target/release/bundle/msi/AIDevHub_0.4.1_x64_en-US.msi.zip.sig" `
  "app-v2/src-tauri/target/release/bundle/nsis/AIDevHub_0.4.1_x64-setup.nsis.zip" `
  "app-v2/src-tauri/target/release/bundle/nsis/AIDevHub_0.4.1_x64-setup.nsis.zip.sig" `
  --clobber
```

---

### Step 8：手动生成 `latest.json`

当前项目必须手动生成 `latest.json`。

格式：

```json
{
  "version": "0.4.1",
  "notes": "发布说明",
  "pub_date": "2026-04-19T23:15:14+08:00",
  "platforms": {
    "windows-x86_64": {
      "signature": "<nsis zip 的 .sig 文件内容>",
      "url": "https://github.com/LZY-Ricardo/AIDevHub/releases/download/app-v0.4.1/AIDevHub_0.4.1_x64-setup.nsis.zip"
    }
  }
}
```

当前已验证可行的做法：

- 读取 `AIDevHub_<version>_x64-setup.nsis.zip.sig` 的完整文本内容
- 填入 `platforms.windows-x86_64.signature`
- `url` 指向同版本 release 下的 `*.nsis.zip`

说明：

- 当前只为 `windows-x86_64` 填了元数据
- 如果后续需要 Linux/macOS 自动更新，需要继续补对应平台条目

---

### Step 9：上传 `latest.json`

示例：

```powershell
gh release upload app-v0.4.1 "F:/myProjects/AIDevHub/latest.json" --clobber
```

上传后可用下面命令确认：

```powershell
gh release view app-v0.4.1 --json assets
```

应能看到：

- `latest.json`

---

## 5. 最终检查清单

发布完成后，至少确认以下内容：

- `git status` 干净
- `main` 已推送到 `origin/main`
- `gh release view app-vX.Y.Z` 正常
- Release 资产包含：
  - 安装包
  - `.sig`
  - `*.zip`
  - `*.zip.sig`
  - `latest.json`
- `latest.json` 中：
  - `version` 正确
  - `pub_date` 合法
  - `url` 指向正确的 release 资产
  - `signature` 与对应 zip 的 `.sig` 一致

## 7. 当前已知限制

1. `latest.json` 仍需手工生成
2. 当前只覆盖 Windows `windows-x86_64`
3. `com.aidevhub.app` 这个 bundle identifier 以 `.app` 结尾，Tauri 会给出警告，后续建议调整

---

## 8. 建议的后续改进

建议后续补一套自动化发布脚本或 GitHub Action，至少做三件事：

1. 自动提升版本并校验配置一致
2. 自动从 `*.nsis.zip.sig` 生成 `latest.json`
3. 自动上传 release 资产并更新 release notes

如果做完这一步，下次发布就不需要再手工拼 `latest.json` 了。
